GTM & GA4 × Shopify Setup

Full GA4 ecommerce tracking via Shopify Custom Pixels — covering page views, product views, add-to-cart, checkout funnel, and purchases.

✦ Overview & Architecture

Shopify's Custom Pixel approach is the recommended method. It runs in a sandboxed iframe with access to all Customer Events — including secure checkout pages that theme injection cannot reach. It also integrates with Shopify's consent API.

💡
Why Custom Pixel over theme.liquid?
Theme injection misses checkout events (checkout_started, payment, purchase) because Shopify's checkout is hosted on a separate domain. Custom Pixels fire on every page including checkout.

The flow works like this:

1
Shopify fires Customer Events — built-in events like page_viewed, product_added_to_cart, checkout_completed, etc.
2
Custom Pixel subscribes and pushes to dataLayer — your pixel code translates each event into GA4-formatted dataLayer objects.
3
GTM processes the dataLayer — your GTM triggers fire based on event names and forward them to GA4, Meta Pixel, or any other tool.
📐
Recommended setup order: This guide follows the correct build sequence — Data Layer Variables → Triggers → Google Tag → GA4 Event Tags . Creating variables and triggers first means they'll already be available in GTM's dropdown pickers when you configure your tags.

1 Confirm GTM Container Setup

Make sure you have your Container ID on hand (format: GTM-XXXXXXX ).

→
Go to tagmanager.google.com → your account → find your Shopify container → copy the Container ID from the top bar.
⚠️
Important: Make sure your GTM container is set to type Web , not Server. You'll replace GTM-XXXXXXX in the pixel code below with your real ID.

2 Create the Shopify Custom Pixel

In your Shopify Admin, navigate to Settings → Customer events → Add custom pixel .

1
Open Shopify Admin → click Settings (bottom-left gear icon)
2
Click "Customer events" in the left sidebar
3
Click "Add custom pixel" — give it a name like GTM - GA4 Ecommerce
4
Paste the pixel code from Section 3 into the code editor
5
Set "Customer privacy" to Not required — unless you're using a consent management platform. You can restrict this later.
6
Click "Save" then "Connect" — the pixel will now be active on your store.

3 The Custom Pixel Code

Copy the entire block below and paste it into the Shopify Custom Pixel editor. Replace GTM-N5JP83PZ with your container ID if different.

📋
This code handles 11 Shopify Customer Events, maps them to GA4-standard event names, and includes User-ID tracking for logged-in customers — ready to be picked up by GTM triggers in the following sections.
JavaScript — Shopify Custom Pixel 
// ================================================================
// SHOPIFY CUSTOM PIXEL — GTM + GA4 Ecommerce Integration
// GTM Container: GTM-N5JP83PZ  |  GA4: G-CC1CKPKC9W
// ================================================================

const GTM_ID = 'GTM-N5JP83PZ';

// Initialize dataLayer
window.dataLayer = window.dataLayer || [];

// ── User-ID Tracking ─────────────────────────────────────────
// Fires a user_identified event when a customer reaches checkout.
// GTM picks this up and passes user_id to GA4 for cross-device attribution.
let _userId = null;

function identifyUser(checkout) {
  const rawId =
    checkout?.order?.customer?.id ||
    checkout?.customer?.id;

  if (!rawId || _userId === rawId) return;

  _userId = rawId;

  const cleanId = String(rawId).replace('gid://shopify/Customer/', '');

  window.dataLayer.push({
    event: 'user_identified',
    user_id: cleanId
  });
}

// ── Load GTM ──────────────────────────────────────────────────
if (!window.gtmLoaded) {
  window.gtmLoaded = true;

  (function(w, d, s, l, i) {
    w[l] = w[l] || [];
    w[l].push({ 'gtm.start': new Date().getTime(), event: 'gtm.js' });
    var f = d.getElementsByTagName(s)[0];
    var j = d.createElement(s);
    var dl = l != 'dataLayer' ? '&l=' + l : '';
    j.async = true;
    j.src = 'https://www.googletagmanager.com/gtm.js?id=' + i + dl;
    f.parentNode.insertBefore(j, f);
  })(window, document, 'script', 'dataLayer', GTM_ID);
}

// ── Helper: Format checkout line items into GA4 items array ──
function formatLineItems(lineItems) {
  return (lineItems || []).map((item, index) => ({
    item_id:      item.variant?.sku || item.variant?.id || item.id,
    item_name:    item.title || item.product?.title,
    item_variant: item.variant?.title,
    item_brand:   item.vendor || item.product?.vendor,
    price:        parseFloat(item.variant?.price?.amount || 0),
    quantity:     item.quantity || 1,
    index:        index
  }));
}

// ── Page Viewed ──────────────────────────────────────────────
analytics.subscribe('page_viewed', (event) => {
  window.dataLayer.push({
    event:         'page_view',
    page_location: event.context.window.location.href,
    page_title:    event.context.document.title,
    page_referrer: event.context.document.referrer
  });
});

// ── Collection / Category Viewed ────────────────────────────
analytics.subscribe('collection_viewed', (event) => {
  const col = event.data.collection;
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:          'view_item_list',
    ecommerce: {
      item_list_id:   col.id,
      item_list_name: col.title,
      items: (col.productVariants || []).map((v, i) => ({
        item_id:      v.sku || v.id,
        item_name:    v.product?.title,
        item_variant: v.title,
        item_brand:   v.product?.vendor,
        price:        parseFloat(v.price?.amount || 0),
        index:        i
      }))
    }
  });
});

// ── Search Submitted ─────────────────────────────────────────
analytics.subscribe('search_submitted', (event) => {
  window.dataLayer.push({
    event:       'search',
    search_term: event.data.searchResult.query
  });
});

// ── Product Viewed ───────────────────────────────────────────
analytics.subscribe('product_viewed', (event) => {
  const v = event.data.productVariant;
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:     'view_item',
    ecommerce: {
      currency: v.price?.currencyCode,
      value:    parseFloat(v.price?.amount || 0),
      items: [{
        item_id:      v.sku || v.id,
        item_name:    v.product?.title,
        item_variant: v.title,
        item_brand:   v.product?.vendor,
        price:        parseFloat(v.price?.amount || 0),
        quantity:     1
      }]
    }
  });
});

// ── Add to Cart ──────────────────────────────────────────────
analytics.subscribe('product_added_to_cart', (event) => {
  const line = event.data.cartLine;
  const v    = line.merchandise;
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:     'add_to_cart',
    ecommerce: {
      currency: line.cost?.totalAmount?.currencyCode,
      value:    parseFloat(line.cost?.totalAmount?.amount || 0),
      items: [{
        item_id:      v.sku || v.id,
        item_name:    v.product?.title,
        item_variant: v.title,
        item_brand:   v.product?.vendor,
        price:        parseFloat(v.price?.amount || 0),
        quantity:     line.quantity
      }]
    }
  });
});

// ── Remove from Cart ─────────────────────────────────────────
analytics.subscribe('product_removed_from_cart', (event) => {
  const line = event.data.cartLine;
  const v    = line.merchandise;
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:     'remove_from_cart',
    ecommerce: {
      currency: line.cost?.totalAmount?.currencyCode,
      value:    parseFloat(line.cost?.totalAmount?.amount || 0),
      items: [{
        item_id:      v.sku || v.id,
        item_name:    v.product?.title,
        item_variant: v.title,
        price:        parseFloat(v.price?.amount || 0),
        quantity:     line.quantity
      }]
    }
  });
});

// ── Cart Viewed ──────────────────────────────────────────────
analytics.subscribe('cart_viewed', (event) => {
  const cart = event.data.cart;
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:     'view_cart',
    ecommerce: {
      currency: cart.cost?.totalAmount?.currencyCode,
      value:    parseFloat(cart.cost?.totalAmount?.amount || 0),
      items:    (cart.lines || []).map((line, i) => ({
        item_id:      line.merchandise?.sku || line.merchandise?.id,
        item_name:    line.merchandise?.product?.title,
        item_variant: line.merchandise?.title,
        price:        parseFloat(line.merchandise?.price?.amount || 0),
        quantity:     line.quantity,
        index:        i
      }))
    }
  });
});

// ── Checkout Started ─────────────────────────────────────────
analytics.subscribe('checkout_started', (event) => {
  const co = event.data.checkout;
  identifyUser(co);
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:           'begin_checkout',
    // ── Custom params (top-level so DLV variables can read them) ──
    customer_type:   (co.order?.customer?.id || co.customer?.id) ? 'returning' : 'new',
    coupon_code:     co.discountApplications?.[0]?.title || '',
    ecommerce: {
      currency: co.currencyCode,
      value:    parseFloat(co.totalPrice?.amount || 0),
      coupon:   co.discountApplications?.[0]?.title || '',
      items:    formatLineItems(co.lineItems)
    }
  });
});

// ── Shipping Info Submitted ──────────────────────────────────
analytics.subscribe('checkout_shipping_info_submitted', (event) => {
  const co = event.data.checkout;
  identifyUser(co);
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:           'add_shipping_info',
    // ── Custom params ──
    customer_type:   (co.order?.customer?.id || co.customer?.id) ? 'returning' : 'new',
    shipping_method: co.shippingLine?.title || '',
    coupon_code:     co.discountApplications?.[0]?.title || '',
    ecommerce: {
      currency:      co.currencyCode,
      value:         parseFloat(co.totalPrice?.amount || 0),
      shipping_tier: co.shippingLine?.title || '',
      items:         formatLineItems(co.lineItems)
    }
  });
});

// ── Payment Info Submitted ───────────────────────────────────
analytics.subscribe('payment_info_submitted', (event) => {
  const co = event.data.checkout;
  identifyUser(co);
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:           'add_payment_info',
    // ── Custom params ──
    customer_type:   (co.order?.customer?.id || co.customer?.id) ? 'returning' : 'new',
    shipping_method: co.shippingLine?.title || '',
    coupon_code:     co.discountApplications?.[0]?.title || '',
    ecommerce: {
      currency: co.currencyCode,
      value:    parseFloat(co.totalPrice?.amount || 0),
      coupon:   co.discountApplications?.[0]?.title || '',
      items:    formatLineItems(co.lineItems)
    }
  });
});

// ── Purchase / Order Completed ───────────────────────────────
analytics.subscribe('checkout_completed', (event) => {
  const co = event.data.checkout;
  identifyUser(co);
  window.dataLayer.push({ ecommerce: null });
  window.dataLayer.push({
    event:           'purchase',
    // ── Custom params (top-level so DLV variables can read them) ──
    customer_type:   (co.order?.customer?.id || co.customer?.id) ? 'returning' : 'new',
    shipping_method: co.shippingLine?.title || '',
    coupon_code:     co.discountApplications?.[0]?.title || '',
    ecommerce: {
      transaction_id: (co.order?.id || co.token || '').replace('gid://shopify/Order/', ''),
      value:          parseFloat(co.totalPrice?.amount || 0),
      tax:            parseFloat(co.totalTax?.amount || 0),
      shipping:       parseFloat(co.shippingLine?.price?.amount || 0),
      currency:       co.currencyCode,
      coupon:         co.discountApplications?.[0]?.title || '',
      items:          formatLineItems(co.lineItems)
    }
  });
});

4 Create Data Layer Variables

Create these before creating tags — they'll be available in GTM's variable picker when you configure event parameters.

How to Create Each Variable

1
Go to Variables → User-Defined Variables → New
2
Click "Variable Configuration" and choose the type: Data Layer Variable — it's under the "Page Variables" category in the type picker
3
Set the "Data Layer Variable Name" — this must match exactly what's pushed to the data layer by your Custom Pixel code (see table below)
4
Leave "Data Layer Version" set to Version 2 — this is the default and correct setting for GTM
5
Name the variable using the DLV - prefix (e.g., DLV - page_location ) and click Save
ℹ️
Variable type: Data Layer Variable. Every variable in this section uses the same type — Data Layer Variable (found under Page Variables in GTM's type picker). This type reads values from the window.dataLayer object that your Custom Pixel code pushes to.

Required Variables

These are used by the Page View tag, Search tag, and the Google Tag's configuration parameter:

Variable Name (in GTM) Variable Type Data Layer Variable Name Used By
DLV - page_location Data Layer Variable page_location GA4 - Page View tag
DLV - page_title Data Layer Variable page_title GA4 - Page View tag
DLV - page_referrer Data Layer Variable page_referrer GA4 - Page View tag
DLV - search_term Data Layer Variable search_term GA4 - Search tag
DLV - user_id Data Layer Variable user_id Google Tag (config parameter)

Optional: Custom Ecommerce Variables

The pixel code in Step 3 already pushes these values to the data layer on checkout events. Create these variables so your GA4 Event Tags can read them (see Section 8 for how to add them as event parameters).

Variable Name (in GTM) Variable Type Data Layer Variable Name Use Case

💡
Naming convention: Prefixing all variables with DLV - makes them easy to identify and find in GTM's variable picker. The "Data Layer Variable Name" column must match exactly what's pushed to the data layer by your Custom Pixel code.

6 Create the Google Tag

The Google Tag (formerly called "GA4 Configuration") initializes the GA4 measurement session inside GTM. In this setup it fires on your custom Shopify events and carries global parameters like user_id , while send_page_view is disabled so your dedicated GA4 Page View tag is the only thing sending page_view .

1
GTM → Tags → New → Click "Tag Configuration" and choose the type: Google Tag (under the "Google" category)
2
Enter your Measurement ID: G-XXXXXXXXXX (from GA4 Admin → Data Streams)
3
Under "Configuration Settings" → click "Add parameter" to add a new Configuration Parameter row:
Configuration Parameter Value Purpose
user_id {{DLV - user_id}} Cross-device user stitching in GA4
traffic_type {{Traffic Type}} Flags internal traffic for filtering (see internal traffic filtering below)
send_page_view false Prevents duplicate page_view events when your dedicated GA4 Page View tag also fires
debug_mode {{Debug Mode}} Allows events to appear in GA4 DebugView without publishing temporary changes to the container.
4
Add two triggers: CE - Page View (fires on every page) AND CE - User Identified (re-fires when a user is identified mid-session at checkout)
5
Save the tag
💡
Why set user_id here instead of on individual tags? Setting it as a Configuration Parameter on the Google Tag means it automatically gets sent with every event after the user is identified. You don't have to add it to each GA4 Event Tag individually. The same pattern works for traffic_type , while send_page_view = false keeps the Google Tag from double-firing page views.
⚠️
Don't use the built-in "All Pages" trigger. Since GTM loads inside the Custom Pixel sandbox, using the custom page_view event trigger ( CE - Page View ) is more reliable than the built-in All Pages trigger. Keeping the Google Tag on CE - Page View plus CE - User Identified also lets it re-read user_id mid-session.
ℹ️
Enable Reporting Identity in GA4: After adding the user_id, go to GA4 Admin → Reporting Identity and set it to "Blended" or "Observed" — otherwise GA4 won't use the User-ID for cross-device reporting even if it's being sent.

9 User-ID Tracking

User-ID lets GA4 stitch together sessions across devices for the same customer. It unlocks login-based reporting, repeat buyer analysis, and accurate user counts. The pixel code and Google Tag setup from previous steps already handle this — here's a summary of how it all connects.

ℹ️
How it works: When a customer reaches checkout, Shopify can expose a customer ID on the checkout object. The pixel extracts that ID, strips Shopify's GID prefix, pushes a user_identified event to the dataLayer, and GTM forwards the user_id field to GA4 on every subsequent hit in that session.

What's Already Set Up

✓
Pixel code (Step 3) — The identifyUser() function is included and called on every checkout event. It pushes user_identified with user_id only — no email is exposed in the data layer.
✓
Data Layer Variables (Step 4) — DLV - user_id is created to read from the data layer.
✓
Trigger (Step 5) — CE - User Identified trigger fires on the user_identified event.
✓
Google Tag (Step 6) — user_id is set as a configuration parameter on the Google Tag, and both CE - Page View and CE - User Identified are attached as triggers.

How identifyUser() Works

This function is called at the top of each checkout event subscriber ( checkout_started , checkout_shipping_info_submitted , payment_info_submitted , checkout_completed ). It extracts the customer ID from checkout.order.customer.id or checkout.customer.id , strips Shopify's GID prefix, and pushes a user_identified event. The _userId guard prevents duplicate pushes within the same session.

💡
Test it: Place a test order, then check GA4 DebugView. You should see a user_identified event with a user_id parameter, and subsequent events in that session should also carry the user_id .

10 Testing Your Setup

Use these two methods to verify everything is firing correctly before calling it done.

⚠️
GTM Preview Mode does not work with Custom Pixels. Because Custom Pixels run in a sandboxed iFrame, GTM's Tag Assistant cannot connect to the container. Do not waste time trying to get it working — use the two methods below instead.

Method A — GA4 DebugView (Recommended)

This is the most reliable way to verify events end-to-end, from your Custom Pixel all the way through GTM and into GA4.

1
In your Google Tag (Step 6), temporarily add a Configuration Parameter: Parameter Name debug_mode → Value true
2
Publish the GTM container
3
Browse your store — trigger a page view, view a product, add to cart, start checkout, complete a test purchase
4
In GA4 → Admin → DebugView — you should see events appear in real-time within 1–2 minutes
5
Click each event — verify the parameters (items array, value, currency, transaction_id, customer_type, shipping_method, coupon_code) are populated correctly
⚠️
Remember to remove debug_mode from your Google Tag and re-publish before going live. Leaving it on will flood DebugView and may affect data collection. This is a temporary testing parameter only.

Method B — Browser Console (dataLayer Inspection)

Use this to inspect the raw data layer pushes before they reach GA4. Useful for debugging data structure issues.

1
Open Chrome DevTools (F12 or right-click → Inspect)
2
In the Console tab, find the context dropdown — it's at the top of the Console and defaults to top . Click it to see a list of all iFrame contexts.
3
Look for entries starting with web-pixel-sandbox-CUSTOM- — there may be multiple. Your GTM pixel is the one with a numeric ID (not "shopify"). Select it.
4
Type window.dataLayer and press Enter. You should see an array of event objects.

Useful console commands once you're in the right context:

JavaScript — Console 
// View all dataLayer pushes so far
window.dataLayer

// Filter for ecommerce events only
window.dataLayer
  .filter(e => e.ecommerce)
  .forEach(e => console.log(e.event, e.ecommerce))

// Check for custom params on checkout events
window.dataLayer
  .filter(e => e.customer_type)
  .forEach(e => console.log(e.event, {
    customer_type:   e.customer_type,
    shipping_method: e.shipping_method,
    coupon_code:     e.coupon_code
  }))
💡
How to identify the right iFrame: When you select the correct CUSTOM context and type window.dataLayer , the first entry should be {gtm.start: ..., event: 'gtm.js'} containing your GTM container ID. If you see an empty array or no dataLayer, try the other CUSTOM context.

  • add_shipping_info fires after shipping method is selected
  • add_payment_info fires after payment method is entered
  • purchase fires on the order confirmation page with correct transaction_id , value , tax , and shipping
  • user_identified fires during checkout and user_id appears in GA4 DebugView
  • Custom params ( customer_type , shipping_method , coupon_code ) appear on checkout/purchase events
  • No duplicate events (check GA4 DebugView for double page_view or purchase events)
  • Google Analytics disconnected from Google & YouTube app (prevents duplicates)
  • debug_mode removed from Google Tag before final publish
  • GTM container version is published

✦ GA4 Admin Setup

Your GTM tags are sending data — but GA4 needs to be configured to receive and report on that data properly. Complete these steps in GA4 Admin before expecting to see custom data in reports.

1. Register Custom Dimensions

Custom event parameters like customer_type , shipping_method , and coupon_code will not appear in GA4 reports until you register them as custom dimensions. Without this step, GA4 receives the data but has nowhere to display it.

1
Go to GA4 Admin → Custom Definitions → Custom Dimensions
2
Click "Create custom dimension"
3
Fill in each dimension using the table below — the "Event parameter" field must match exactly what your GTM tags send
Dimension Name (display label) Scope Event Parameter Description
Customer Type Event customer_type Known vs. guest customer status
Shipping Method Event shipping_method Delivery method selected
Coupon Code Event coupon_code Discount code applied
Search Term Event search_term Site search query
⚠️
GA4 has a limit of 50 custom event-scoped dimensions on standard properties (and 125 on 360). Check your current usage before adding new ones. Custom dimensions only apply to data collected after they are created — they are not retroactive.

2. Enable Reporting Identity (User-ID)

This tells GA4 to use the user_id you're sending for cross-device session stitching.

1
Go to GA4 Admin → Reporting Identity
2
Select "Blended" (recommended) — this uses User-ID when available, then Google Signals, then Device-ID as fallbacks
3
Click Save
ℹ️
"Blended" vs "Observed": Blended gives GA4 the most flexibility to de-duplicate users. "Observed" is stricter and only uses User-ID or Google Signals. Either works — "Blended" is the safer default.

3. Define Internal Traffic

Filter out your team's browsing so it doesn't skew reports.

1
Go to GA4 Admin → Data Streams → your stream → Configure Tag Settings → Show More → Define Internal Traffic
2
Click "Create" and add a rule — name it (e.g., "Office IP"), set traffic_type value to internal , and enter your office IP addresses or CIDR range
3
Go to GA4 Admin → Data Settings → Data Filters
4
Find the "Internal Traffic" filter and change it from Testing to Active
💡
Leave the filter in "Testing" mode first to verify it's catching the right traffic. In GA4 reports, add a comparison for Test data filter name = "Internal Traffic" to preview what would be excluded. Once you're confident it's correct, switch to "Active."

4. Disable Enhanced Measurement (Selective)

GA4's Enhanced Measurement auto-collects certain events from the main page context. Some of these duplicate what your Custom Pixel already tracks and should be turned off. Others provide useful data that your Custom Pixel can't track from the sandbox, and should stay on.

1
Go to GA4 Admin → Data Streams → your stream
2
Click the gear icon next to "Enhanced measurement"
3
Turn off these three — they duplicate events your Custom Pixel already handles:
Setting Action Why
Page views 🔴 Turn OFF Your Custom Pixel fires page_view — this would double-count
Scrolls 🔴 Turn OFF Fires from the main page but the sandbox can't correlate it properly — noisy data
Site search 🔴 Turn OFF Your Custom Pixel fires search with search_term — this would create duplicate view_search_results events
Outbound clicks 🟢 Leave ON Your Custom Pixel doesn't track outbound clicks — useful free data
Form interactions 🟢 Leave ON Your Custom Pixel doesn't track forms — useful if you have a contact or quote request form
Video engagement 🟢 Leave ON Auto-tracks YouTube embeds — no conflict with your ecommerce tracking
File downloads 🟢 Leave ON Link click triggers don't work in the sandbox, so this is your only way to track file downloads
⚠️
At minimum, turn off "Page views", "Scrolls", and "Site search" — these will definitely duplicate with your GTM Custom Pixel setup and inflate your event counts.

5. Add Unwanted Referrals for Shopify Checkout

Shopify checkout can route users through additional domains like checkout.shopify.com and shop.app . Add these as unwanted referrals so GA4 keeps attribution on the original traffic source instead of overwriting sessions mid-checkout.

1
Go to GA4 Admin → Data Streams → your stream → Configure Tag Settings → Show More → List unwanted referrals
2
Add these domains — checkout.shopify.com , shop.app , and pay.shopify.com
3
Save your changes — this preserves attribution across the checkout handoff
ℹ️
Do not add Shopify checkout domains to cross-domain measurement. For this setup, the unwanted referral list is the correct fix.

6. Mark Key Events (formerly "Conversions")

Tell GA4 which events are business-critical so they appear in key event reports and can be imported into Google Ads. GA4 renamed "Conversions" to "Key events" in 2024.

1
Go to GA4 Admin → Events (wait 24–48 hours after setup for events to appear here)
2
You'll see two tabs: "Key events" and "Recent events" — the Key events tab shows events already marked. Click "Recent events" to find and mark new ones.
3
Click the star icon (☆) next to each event to toggle it as a key event:
Event Mark as Key Event? Why
purchase ★ Yes Revenue tracking — this is your primary key event
begin_checkout ★ Yes Checkout initiation — critical for funnel analysis and Google Ads optimization
add_to_cart ★ Recommended Useful for Google Ads mid-funnel bidding signals
add_payment_info Optional Useful for deep funnel analysis
view_item Optional Only if you want to optimize Google Ads for product page views
💡
If you're running Google Ads: Marking purchase and begin_checkout as key events lets you import them into Google Ads for Smart Bidding optimization. This is essential for getting the most out of your ad spend.
ℹ️
Events not showing up? It can take 24–48 hours after an event first fires for it to appear in the Events list. If you just set up your tracking, come back tomorrow and check the "Recent events" tab again.

GA4 Admin Checklist

  • Custom dimensions registered: customer_type , shipping_method , coupon_code , search_term
  • Reporting Identity set to "Blended" (or "Observed")
  • Internal traffic rule created with office IP addresses
  • Internal traffic data filter set to "Testing" (then "Active" once verified)
  • Enhanced Measurement: Page views, Scrolls, and Site search turned off — Outbound clicks, Forms, Video, File downloads left on
  • Unwanted referrals added: checkout.shopify.com , shop.app , pay.shopify.com
  • purchase marked as key event
  • begin_checkout marked as key event
  • add_to_cart marked as key event (recommended)

✦ GA4 Metrics Guide

Where to find each key business metric in GA4 once your tracking is fully set up:

Active Users (Logged-in)

→
Reports → User → User acquisition — Filter by user_id is set to isolate only logged-in sessions. GA4 de-duplicates users across devices using the user_id you're now sending.

Regular Buyers (2+ Purchases / 4 Weeks)

→
Explore → Cohort Exploration — Set cohort inclusion to purchase , returning condition to purchase , within a 28-day window. Or use Segment Builder : create a user segment where event_count for purchase ≥ 2 within 28 days.

Average Cart Size

→
Reports → Monetization → Ecommerce Purchases — The Average purchase revenue metric is built in. For average items per order, use Explore with a custom calculated metric: items_purchased ÷ transactions .

Cart Abandonment %

→
Explore → Funnel Exploration — Build a funnel with these steps: add_to_cart → begin_checkout → purchase . GA4 shows the percentage drop-off at each step automatically. The gap between add_to_cart and purchase is your abandoned cart rate.

Individual Transaction Drill-Down

→
Reports → Monetization → Ecommerce Purchases — Click any Transaction ID in the table to see a full breakdown: every item, quantity, unit price, revenue, tax, and shipping for that specific order.

Coupon Code Performance

→
Explore → Free Form Exploration — Add Coupon Code (your custom dimension) as a row, and Purchase revenue , Transactions , and Average purchase revenue as values. This shows you which discount codes drive the most revenue, how often they're used, and whether they increase or decrease average order value.

New vs. Returning Customers

→
Explore → Free Form Exploration — Add Customer Type (your custom dimension) as a row, with Purchase revenue , Transactions , and Average purchase revenue as values. Compare new vs. returning buyer behavior — are returning customers spending more per order? What percentage of revenue comes from repeat buyers?

Shipping Method Preferences

→
Explore → Free Form Exploration — Add Shipping Method (your custom dimension) as a row, with Transactions and Purchase revenue as values. See which shipping options customers choose most, and whether offering faster shipping correlates with higher order values.

Coupon Impact on Checkout Completion

→
Explore → Funnel Exploration — Build a funnel ( begin_checkout → purchase ) and add Coupon Code as a breakdown dimension. This reveals whether customers using discount codes complete checkout at higher rates than those who don't — a key insight for deciding whether to promote coupons more aggressively.
📊
Pro tip — save your Explorations. GA4 Explorations (funnels, cohorts, segments) don't appear in standard reports. Build each one, then click the three-dot menu and "Share exploration" with your team so everyone can access the same views without rebuilding them.

✦ Google & YouTube App Settings

If your Shopify store has the Google & YouTube app installed, keep it for product feed sync and Google Ads connectivity — but configure it carefully to avoid duplicate tracking with your Custom Pixel + GTM setup.

What the App Does

The Google & YouTube app can connect multiple Google services to your Shopify store. Each serves a different purpose, and not all of them conflict with your GTM setup:

Connected Service What It Does Conflicts with GTM?
Google Ads Powers product feed sync for Google Shopping and Merchant Center free listings 🟢 No conflict — keep connected
Google Analytics Installs its own GA4 measurement tag that sends page views and ecommerce events 🔴 Duplicates your Custom Pixel tracking — disconnect
Conversion measurement Adds Google Ads and GA4 conversion tags managed by the app 🔴 Duplicates your GTM conversion tracking — leave off
Customer Match Builds audience lists from customer data for Google Ads targeting 🟢 No conflict — can enable later

Recommended Settings

1
Go to Shopify Admin → Google & YouTube → Settings
2
Google Ads → Keep connected — this powers your product feed for Shopping campaigns and free listings. It does not conflict with your GTM ecommerce tracking.
3
Google Analytics → Disconnect — your Custom Pixel + GTM setup is more comprehensive than what the app provides. Keeping GA4 connected here will create duplicate page_view , purchase , and other events.
4
Conversion measurement → Leave OFF — since you're handling conversion tracking through your Custom Pixel → GTM → GA4 pipeline, turning this on creates duplicate conversion events.
5
Customer Match → Leave OFF for now — this is safe to enable later once your base tracking is verified and stable. It automatically builds audience lists from customer data for Google Ads.
⚠️
Don't disconnect Google Ads. Even though you're tracking conversions through GTM, the Google Ads connection in this app handles your product feed sync for Google Shopping / Merchant Center. Disconnecting it will break your Shopping campaigns and free product listings.
💡
Enhanced Conversions: If you later want Enhanced Conversions for Google Ads (hashed email/phone data for better conversion match rates), the Google & YouTube app handles this more seamlessly than GTM in the Custom Pixel sandbox. You can revisit the Conversion measurement toggle at that point — but test carefully for duplicate events in GA4 DebugView before leaving it on.

After Disconnecting GA4

Once you disconnect Google Analytics from the app, verify in GA4 DebugView that you're not seeing duplicate events. You should see one page_view per page load and one purchase per order — not two of each. If you still see duplicates, check for legacy GA4 tags in your Shopify theme files or other Shopify apps that might be injecting tracking.

Google & YouTube App Checklist

  • Google Ads — connected (for product feed / Shopping)
  • Google Analytics — disconnected (Custom Pixel handles this)
  • Conversion measurement — OFF
  • Customer Match — OFF for now (enable once tracking is stable)
  • Verified no duplicate events in GA4 DebugView after changes

✦ Third-Party Scripts: Where They Go

Shopify's Custom Pixel runs in a sandboxed iFrame, which means some scripts work inside it and others don't. The deciding factor is whether the script needs DOM access (the ability to see, read, or modify elements on the actual page).

The Rule

→
Does it just need to know what happened? (user bought something, viewed a product, added to cart) → GTM via Custom Pixel — it only needs data layer events
→
Does it need to see or touch the page? (record a session, show a widget, read form fields, inject UI) → theme.liquid — it needs full DOM access

Scripts That Go in theme.liquid (Require DOM Access)

Add these directly to your Shopify theme: Online Store → Themes → Edit Code → theme.liquid , just before the closing </head> tag.

Script Why It Needs DOM Notes
ZoomInfo Visitor identification, form enrichment Reads page content and injects into form fields
Hotjar Session recording, heatmaps Records the entire DOM and user interactions
Microsoft Clarity Session recording, heatmaps Same as Hotjar — needs to observe DOM changes
Crazy Egg Heatmaps, click tracking Overlays click data on actual page elements
Lucky Orange Session replay, heatmaps Records DOM state and user interactions
Drift / Intercom / LiveChat Chat widgets Renders a visible widget on the page
OptinMonster / Privy Popups, overlays, banners Injects modal/popup elements into the DOM
Accessibility tools Widget overlays Tools like AccessiBe, UserWay that modify the page
ℹ️
theme.liquid limitation: Scripts added here will fire on every page except checkout. For most of these tools (session recording, chat widgets, visitor ID), that's fine — they don't need checkout page access. Checkout is on a separate Shopify-hosted domain.

Scripts That Go in GTM via Custom Pixel (Event-Based Only)

These fire based on data layer events and don't need to see the page. Add them as tags in GTM using the same Custom Event triggers you created in Step 5.

Script What It Tracks GTM Tag Type
Meta Pixel (Facebook) Purchase, Add to Cart, View Content, etc. Custom HTML or Meta Pixel template
TikTok Pixel Purchase, Add to Cart, View Content Custom HTML or TikTok template
Pinterest Tag Checkout, Add to Cart, Page Visit Custom HTML or Pinterest template
Snapchat Pixel Purchase, Add to Cart Custom HTML
Reddit Pixel Purchase, Add to Cart, Page Visit Custom HTML
Klaviyo Ecommerce events for email flows Custom HTML
Google Ads conversions Purchase, Begin Checkout Google Ads Conversion Tracking tag
Bing/Microsoft Ads Purchase, Add to Cart Custom HTML or UET tag template
💡
No pixel code changes needed. Since all these event-based pixels read from the same data layer that your Custom Pixel code already populates, you just create new GTM tags that fire on the existing Custom Event triggers (CE - Purchase, CE - Add to Cart, etc.). The Shopify pixel code stays exactly the same.

Quick Decision Guide

When evaluating a new script for a Shopify site, ask these questions:

1
Does it render something visible on the page? (chat bubble, popup, overlay, widget) → theme.liquid
2
Does it record or observe user behavior on the page? (session replay, heatmap, click tracking) → theme.liquid
3
Does it read or modify form fields? (form enrichment, auto-fill, validation) → theme.liquid
4
Does it only fire on specific events? (purchase, add to cart, page view) → GTM via Custom Pixel
5
Not sure? Check the script's documentation for mentions of "DOM", "element", "selector", "widget", or "inject". If you see those → theme.liquid . If the docs only mention "events", "conversions", or "pixels" → GTM via Custom Pixel
⚠️
Never add the same script in both places. If a script is in theme.liquid AND in a GTM tag that fires from the Custom Pixel, you'll get duplicate tracking. Pick one location per script.
